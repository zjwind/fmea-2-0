# FMEA 2.0 功能规格文档（V3）

> 版本：V3.0　适用范围：与 `fmea-prototype-v3` 原型一致的裁剪后系统范围
> 编制依据：《FMEA2.0-需求分析文档》《系统主页及菜单说明》《fmea_scenario_extracted（用户场景）》《UI风格要求》
> 表达方式：各功能模块均以表格表达（功能项表 + 验收标准表）

## 1. 文档说明

### 1.1 范围说明

本版本（V3）依据《系统主页及菜单说明》对系统进行裁剪，仅保留与 `fmea-prototype-v3` 原型一致的功能集合。相对于早期全量设计（13 大模块），V3 将"评审/审批"合并进 DRBFM 分析任务的流程步骤，将"基线输出"嵌入失效分析与审批后流程，将"知识库/基线库/措施库"作为预留入口，移除独立的"项目管理 / 入库管理 / 迭代更新 / 看板统计 / 系统配置"顶层菜单，统一收敛为下表菜单。

| 一级菜单 | 二级菜单 | 模块归类 | 说明 |
|---|---|---|---|
| 首页 | — | 基础模块 | 工作台看板，待办、进度、风险分布 |
| 评估任务 | 系统级评估 / 部件级评估 | 关键模块 | DRBFM 触发评估 |
| DRBFM 分析任务 | 我负责的 / 全部 | 关键模块 | 结构分析→功能分析→失效分析→评审→审批 |
| 评审意见管理 | 我提出的 / 待我处理的 | 基础模块 | 评审意见跟踪与闭环 |
| 知识库 | 基线库 / 措施库 / 历史问题库 | 基础模块 | 基线库、措施库预留；历史问题库可用 |
| 系统菜单 | SOD 评价标准 | 基础模块 | S/O/D 评分标准与 AP 判定查询 |

### 1.2 角色定义

| 角色 | 英文/简称 | 主要职责 |
|---|---|---|
| 技术系统工程师 | TSE | 发起评估、维护变更清单、负责部件级分析 |
| 产品系统工程师 | PSE | 系统级分析负责人、评审、审批 |
| 协作人 | Collaborator | 部件级评估/分析的协同填写人 |
| 评审专家 | Reviewer | PSE / 专业组长 / 质量经理 / DFX SE 等评审角色 |
| 审批人 | Approver | 分析结论审批（PSE / 上级） |

### 1.3 通用约定

| 约定项 | 说明 |
|---|---|
| 项目数据来源 | 项目基本信息、成员、权限由 PMS 同步，本系统不维护项目主数据 |
| 飞书集成 | 结构框图（画板）、功能矩阵（电子表格）、失效分析表、评估表等以飞书资源嵌入/链接，预留连接字段 |
| AI 集成 | 失效模式/原因/措施由 AI 生成至临时区，用户采纳后入库；SOD 由后端计算、AI 给建议值 |
| 风险等级 | H（高）/ M（中）/ L（低），用于结构节点、功能、AP 判定的颜色标记 |
| 五维评分取值 | 高=10 / 中=5 / 低=3 / 没有=0；风险分边界：≥8 为 H，≥5 且 <8 为 M，<5 为 L |

## 3. 评估任务模块（关键模块 · DRBFM 触发评估）

### 3.1 评估任务列表

| 编号 | 功能项 | 描述 | 操作 |
|---|---|---|---|
| EVAL-01 | 系统级/部件级切换 | 通过菜单或 `?level=system\|part` 区分两类评估列表 | 切换查看 |
| EVAL-02 | 任务列表 | 展示项目、里程碑(TR)、名称、TSE、PSE、状态、起止时间 | 排序/筛选 |
| EVAL-03 | 状态标识 | 待开始 / 进行中 / 协作人签核中 / 待审核 / 已完成 | 颜色标签 |
| EVAL-04 | 新建评估 | 选择 PMS 项目与里程碑，创建评估任务 | 新建 |
| EVAL-05 | 进入评估流程 | 点击进入多步骤评估工作流 | 跳转 |

### 3.2 评估工作流（多步骤）

系统级步骤：导入变更清单 → 质量策划匹配 → 历史问题加载 → 五维评估与评分 → 复核确认。
部件级步骤：在五维评估前增加"指派协作人"步骤（协作人填写部件级评估，TSE 复核提交，**部件级无需协作人签核**，TSE 直接提交）。

| 编号 | 步骤 | 功能描述 | 飞书/AI/集成 | 输出 |
|---|---|---|---|---|
| EVAL-10 | 导入变更清单 | 维护多级结构变更项（结构名、开发类型、IMN、变更点、质量匹配） | 可导入文件 | 变更清单 |
| EVAL-11 | 质量策划匹配 | 上传/匹配质量策划内容，关联变更项 | 文件解析 | 质量匹配结果 |
| EVAL-12 | 历史问题加载 | 按 BG/域加载历史问题库相关问题供参考 | 历史问题库 | 历史问题列表 |
| EVAL-13 | 五维评估与评分 | 按技术新颖度、影响范围、严重度、变更复杂度、历史问题五维选择高/中/低/没有，后端计算风险分与等级，给出 DRBFM 建议 | 评估表（飞书电子表格，预留连接字段） | 风险分/等级/DRBFM 建议 |
| EVAL-14 | 指派协作人（部件级） | TSE 指派协作人分担部件评估 | 通知协作人 | 协作分工 |
| EVAL-15 | 复核确认 | TSE/PSE 复核，确认 DRBFM 结论（是否进入分析），提交 | 触发分析任务生成 | 评估结论 |

### 3.3 五维评分规则

| 维度 | 取值与分值 | 说明 |
|---|---|---|
| 技术新颖度 / 影响范围 / 严重度 / 变更复杂度 / 历史问题 | 高=10 / 中=5 / 低=3 / 没有=0 | 各维度单选 |
| 风险分计算 | 由后端按规则汇总 | 计算逻辑见详细设计 V3 |
| 风险等级 | H：风险分 ≥8；M：≥5 且 <8；L：<5 | 用于 DRBFM 建议 |
| DRBFM 建议 | 高风险建议进入 DRBFM 分析 | 结论由人工确认 |

### 3.4 验收标准

| 编号 | 验收标准 |
|---|---|
| AC-EVAL-01 | 系统级/部件级列表互不串扰，状态与实际流程一致 |
| AC-EVAL-02 | 五维选择后风险分与等级由后端计算返回，前端不自行计算 |
| AC-EVAL-03 | 评估提交后按结论生成对应 DRBFM 分析任务并关联 evaluation_task |
| AC-EVAL-04 | 部件级 TSE 可直接提交，无协作人签核卡点 |
| AC-EVAL-05 | 评估表飞书资源链接可打开，连接字段持久化保存 |
| AC-EVAL-06 | 撤回/重复提交受控（记录 submit_count、is_withdrawn） |

| AN-S3-04 | SOD 评分 | 后端按 SOD 标准计算 S/O/D 与 AP，AI 给出建议值（可修改） | SOD 标准 + AP 参照 | 评分与 AP |

| AN-S4-03 | 评审意见 | 评审人提交评审结论与意见 | — | 评审意见 |
| AN-S4-04 | 意见闭环率 | 统计意见闭环率，作为流转条件 | — | 闭环率 |

### 4.6 步骤五：审批

| 编号 | 功能项 | 描述 | 输出 |
|---|---|---|---|

### 4.7 验收标准

| 编号 | 验收标准 |
|---|---|
| AC-AN-01 | 五步流程状态与任务 status 一致，步骤可前进/回退受控 |
| AC-AN-02 | 结构框图、功能矩阵、失效分析表、会议纪要的飞书连接字段均持久化且可打开 |
| AC-AN-03 | AI 重新生成仅删除未采纳临时数据，已采纳项保留 |
| AC-AN-04 | SOD 与 AP 由后端计算，AI 建议值可被用户覆盖 |
| AC-AN-05 | 评审意见未闭环不可提交审批 |
| AC-AN-06 | 审批通过后自动生成基线及落地项，可被落地跟踪引用 |
